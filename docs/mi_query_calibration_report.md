## Known limitations

**Grouped query + value filter** — evidenced from live behaviour:

- Probe: `balance by Obligor Region (NUTS3) where LTV above 50%`
- Filter parsed into the spec: **False**
- Filter applied (reconciliation): **False** (400 of 400 records)
- Omission surfaced to the user (warning): **False**
- Supported today: **False**

A value filter combined with a grouping (`… by <dim> where <measure> above <x>`) is **not** applied today, and the omission is not yet surfaced as a warning. The supported filtered shape is the ungrouped filtered KPI (`how many loans have LTV above 50%`), verified by the `filtered_kpi` class above. Recommended follow-up: extend the fail-closed guard to filters (warn/refuse when a recognised filter phrase is not applied) and add grouped+filter parsing.
